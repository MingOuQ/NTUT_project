# 基於多光譜影像與深度學習之表面瑕疵智慧檢測系統

## 成果報告

**專題名稱**：基於機器視覺與光電感測之表面瑕疵智慧檢測系統  
**使用平台**：ESP32S / ESP32-CAM / Python (YOLOv8)

## 摘要

本專題設計並實作了一套結合**多光譜影像分析 (Multi-spectral Imaging)**、**YOLOv8 深度學習物件偵測**與**邊緣運算 (Edge Computing)** 的自動化表面瑕疵檢測系統。系統以 ESP32S 微控制器作為硬體控制核心，搭配 ESP32-CAM 進行影像擷取，透過依序切換紅 (R)、綠 (G)、藍 (B) 三色 LED 光源照射待測物表面，利用不同波長光線與表面瑕疵的差異性反射特性，突顯刮痕、髒污、凹陷等微小缺陷。擷取之多光譜影像經 USB Serial 傳輸至 PC 端，由 YOLOv8-nano 深度學習模型進行即時瑕疵辨識與定位，並可彈性切換 OpenCV 傳統視覺演算法作為備援模式。檢測結果透過 OLED 螢幕與蜂鳴器在機台端即時回饋，同時透過 ESP32S 內建的 Wi-Fi AP 熱點與 WebSocket 即時通訊協定，將瑕疵數據同步推播至手機端 PWA 監控儀表板，實現聲光警報與歷史紀錄追蹤。系統另整合 Excel 自動化報表匯出功能，可即時生成含瑕疵座標、類型、AI 信心度等資訊的 `.xlsx` 檢測報告。實測結果顯示，系統可在數秒內完成全流程 RGB 三色光譜掃描與 AI 分析，有效辨識 5 種不同形態的表面瑕疵（point、square、circle、line_h、line_v），並具備模型微調 (Fine-tuning) 機制以應對真實環境的領域落差 (Domain Gap)。

## 1. 前言

在現代製造業中，產品表面瑕疵檢測是品質管控流程中最為關鍵的環節之一。傳統的人工目視檢測方式不僅效率低落、勞動成本高昂，且受限於人眼疲勞與主觀判斷，容易產生漏檢與誤判的情況。隨著人工智慧與嵌入式系統技術的快速發展，導入**自動光學檢測 (Automated Optical Inspection, AOI)** 系統已成為產業升級的必然趨勢。

然而，市面上的工業級 AOI 設備往往造價高昂（動輒數十萬至百萬新台幣），且系統封閉、不易客製化，對於中小型企業或學術研究單位而言門檻過高。因此，本專題旨在以低成本的嵌入式硬體（ESP32 系列微控制器）搭配開源 AI 框架（YOLOv8），建構一套**具備多光譜分析能力、AI 即時推論、無線行動監控**的完整瑕疵檢測解決方案，證明在有限預算下亦可實現接近工業水準的自動化品質檢測能力。

## 2. 研究背景與動機

### 2.1 產業現況與痛點

在 Mini LED、Micro LED 顯示面板製程中，面板上數以萬計的微小 LED 晶粒在大量轉移 (Mass Transfer) 過程中容易產生各種表面瑕疵，如：

| 瑕疵類型 | 特徵描述 | 工業影響 |
|----------|----------|----------|
| **point（斑點）** | 微小點狀雜質或暗點 | 影響顯示均勻度 |
| **square（方塊突起）** | 不規則方塊形狀異物 | 可能導致短路或斷路 |
| **circle（圓形凹陷）** | 圓形壓痕或氣泡 | 影響結構完整性 |
| **line_h（水平線狀瑕疵）** | 水平方向的刮痕或裂痕 | 影響走線完整性 |
| **line_v（垂直線狀瑕疵）** | 垂直方向的刮痕或裂痕 | 同上 |

傳統的單一白光 AOI 系統在面對這些瑕疵時，容易因光線角度固定、對比度不足而漏判。本專題引入**多光譜分析技術**，透過不同波長的光源依序照射待測物，利用瑕疵區域與正常區域在不同光波段下的反射率差異，大幅提升檢測靈敏度。

### 2.2 技術發展趨勢

近年來，深度學習在物件偵測領域取得突破性進展，以 YOLO (You Only Look Once) 系列為代表的即時物件偵測架構，已被廣泛應用於工業品質檢測場景。YOLOv8 作為最新一代架構，在推論速度與檢測精度之間取得了良好平衡，其 nano 版本（YOLOv8n）更適合在算力有限的環境中部署。

與此同時，ESP32 系列微控制器以其低功耗、內建 Wi-Fi/Bluetooth、高速雙核處理器等特性，成為 IoT 與邊緣運算的主流方案。ESP32-CAM 模組更整合了 OV2640 CMOS 影像感測器，可直接進行影像擷取，為低成本視覺檢測系統提供了理想的硬體平台。

### 2.3 研究動機

基於上述背景，本專題的研究動機可歸納為以下三點：

1. **降低 AOI 系統的建置門檻**：以總硬體成本不超過新台幣一千元的方案，實現具備 AI 辨識能力的瑕疵檢測系統。
2. **驗證多光譜分析的實用價值**：透過 RGB 三色光源依序照射的方式，實驗比較多光譜檢測與單一光源檢測的效能差異。
3. **建構完整的 IoT 智慧檢測生態系**：從硬體感測、AI 推論、無線通訊到行動監控，實現端到端 (End-to-End) 的全方位整合。

## 3. 研究目標

本專題的具體研究目標如下：

1. **設計三節點分散式硬體架構**：建構以 ESP32S（控制節點）、ESP32-CAM（影像節點）、PC（運算中樞）為核心的三層式系統架構，各節點分工明確、通訊穩定。

2. **實現多光譜自動檢測流程**：完成紅光 → 綠光 → 藍光依序自動切換拍攝、多通道影像融合分析的全自動檢測序列，支援單面板與 3×3 九宮格拼接面板雙模式。

3. **建立 AI 瑕疵辨識模型**：以 YOLOv8-nano 為基礎，使用自建的 Mini LED 瑕疵資料集（5 類、120 張訓練影像）進行遷移學習，實現 5 種瑕疵類型的精準定位與分類。

4. **開發無線行動監控系統**：實作 ESP32S AP 熱點 + WebSocket + PWA 行動端 Dashboard，使現場操作人員可透過手機即時接收檢測結果、聲光警報與歷史數據。

5. **整合自動化報表功能**：每次檢測完畢自動產出 Excel 格式 (`.xlsx`) 的瑕疵報告，內含座標、類型、信心度等完整資訊，方便品質管理追蹤。

### 4.2 多光譜影像分析原理

#### 4.2.1 光譜選擇性反射

不同材質的表面瑕疵對不同波長的光具有不同的反射率特性。以 Mini LED 面板為例：

- **紅光 (λ ≈ 620 nm)**：對紅色子像素區域的瑕疵最為敏感，暗色污漬在紅色通道下對比度最高。
- **綠光 (λ ≈ 520 nm)**：對綠色子像素區域及表面刮痕的辨識效果最佳。
- **藍光 (λ ≈ 470 nm)**：短波長光線散射能力較強，適合偵測表面微小凸起與粒子狀瑕疵。

透過依序照射三色光源並分別擷取影像，系統可獲得待測物在三個波段下的反射資訊，大幅超越單一白光照明的資訊量。

#### 4.2.2 多通道影像融合

PC 端軟體從紅光影像提取 R 通道、綠光影像提取 G 通道、藍光影像提取 B 通道後，以**取最小值 (Element-wise Minimum)** 的方式進行融合：

```python
# 從各自影像中提取對應通道
r_channel = red_img[:, :, 2]    # BGR 中的 R 通道
g_channel = green_img[:, :, 1]  # G 通道
b_channel = blue_img[:, :, 0]   # B 通道

# 融合：取各通道的最小值（瑕疵在所有通道都是暗的）
combined_gray = np.minimum(np.minimum(r_channel, g_channel), b_channel)
```

此融合策略的物理意義為：真正的表面瑕疵（如刮痕、髒污）在所有波段下都會呈現較低的反射率（較暗），而因特定角度或色彩造成的假陽性則不會在三個通道中同時出現暗區，因此融合後可有效抑制誤判。

### 4.3 YOLOv8 深度學習瑕疵辨識

#### 4.3.1 模型架構

本系統採用 YOLOv8-nano（YOLOv8n）作為瑕疵偵測模型，其主要特點：

| 參數 | 數值 |
|------|------|
| 模型架構 | YOLOv8n (nano) |
| 參數量 | 約 3.2M |
| 模型大小 | 約 6.2 MB |
| 輸入解析度 | 256 × 256 |
| 偵測類別數 | 5 類 |
| 推論速度 | < 50ms / 張 (CPU) |

#### 4.3.2 訓練資料集

使用自建的 `mini_led_defect_dataset_rgb_label_v2` 資料集：

| 項目 | 數量 |
|------|------|
| 訓練影像 | 96 張 |
| 驗證影像 | 24 張 |
| 影像尺寸 | 256 × 256 |
| 瑕疵類別 | 5 種（point、square、circle、line_h、line_v）|
| 每張瑕疵數 | 1 ～ 8 個（隨機分佈）|
| 標註格式 | YOLO Detection |

#### 4.3.3 遷移學習 (Transfer Learning) 與微調 (Fine-tuning)

模型訓練採用兩階段策略：

1. **第一階段 — 合成資料預訓練**：以程式自動生成的合成瑕疵影像進行初步訓練，使模型學習 5 種瑕疵的基本形態特徵。
2. **第二階段 — 真實環境微調**：透過系統內建的「一鍵拍照收集」功能，實際收集真實環境照片，經手動標註後進行 Fine-tuning（預設 300 Epochs），以彌補合成資料與真實場景之間的領域落差 (Domain Gap)。

訓練參數配置：

```python
model.train(
    data="dataset.yaml",
    epochs=300,
    batch=16,
    imgsz=256,
    patience=20,     # Early stopping
    save_period=10,  # 每 10 Epoch 儲存 checkpoint
)
```

### 4.4 OpenCV 傳統視覺檢測（備援模式）

當 AI 模型不可用時，系統自動切換為 OpenCV 傳統演算法模式，處理流程如下：

1. **通道提取**：依據當前 LED 顏色提取對應的 BGR 通道。
2. **高斯模糊**：以 3×3 核進行去雜訊 (`GaussianBlur`)。
3. **自適應閾值二值化**：使用 `adaptiveThreshold` 搭配 Gaussian 加權，自動適應局部亮度差異。
4. **形態學操作**：先開運算 (MORPH_OPEN) 去除小雜訊，再閉運算 (MORPH_CLOSE) 連接破碎輪廓。
5. **輪廓偵測與篩選**：以面積 (10 ～ 50000 px²) 與長寬比進行瑕疵分類：
   - 長寬比 > 4.0 → scratch（劃痕）
   - 長寬比 > 2.0 → crack（裂痕）
   - 面積 > 500 → stain（污漬）
   - 其餘 → spot（斑點）

### 4.5 即時通訊架構

#### 4.5.1 USB Serial 通訊協定

PC 與兩顆 ESP32 之間採用自定義的 ASCII 指令協定（Baud Rate: 115200），主要指令包括：

| 方向 | 指令範例 | 說明 |
|------|----------|------|
| PC → ESP32S | `LED:R` | 切換紅光 |
| PC → ESP32-CAM | `CAPTURE` | 拍攝影像 |
| PC → ESP32S | `RESULT:DEFECT,x,y,w,h,type,conf` | 回報瑕疵 |
| ESP32-CAM → PC | `IMG:<size>` + JPEG 二進位 | 影像資料 |
| ESP32S → PC | `BTN_PRESSED` | 按鈕觸發 |

影像傳輸採用「標頭 + 二進位資料 + 結尾確認」的混合協定：先以 ASCII 發送 `IMG:<size>\n` 告知資料長度，接著傳輸原始 JPEG 二進位資料，最後以 `IMG_OK\n` 確認傳輸完成。

#### 4.5.2 WebSocket 即時廣播

ESP32S 以 AP 模式自建 Wi-Fi 熱點 (`Defect_Detector_WiFi`)，並在 Port 81 啟動 WebSocket 伺服器。當 PC 端將瑕疵檢測結果傳送給 ESP32S 時，ESP32S 會同步將 JSON 格式的數據透過 WebSocket 廣播至所有已連線的手機用戶端：

```json
{
  "status": "DEFECT",
  "count": 3,
  "type": "point",
  "conf": 0.92,
  "x": 128,
  "y": 96
}
```

手機端 PWA Dashboard 以 JavaScript 接收訊息後，即時更新 UI 狀態（掃描中/發現瑕疵/通過）、觸發 Web Audio API 合成警報音效、並記錄歷史日誌。整體延遲低於 50 毫秒。

### 4.6 PWA 行動端監控儀表板

手機端 Dashboard 採用 HTML5 + CSS3 + JavaScript (ES6+) 開發，以 PWA (Progressive Web App) 形式部署在 ESP32S 的內建 HTTP 伺服器上（Port 80），具備以下特點：

- **暗色霓虹科技風格 UI**：glassmorphism 面板、漸層光暈動畫、狀態指示球。
- **三種視覺狀態**：就緒（綠色呼吸燈）→ 掃描中（藍色跑馬燈動畫）→ 瑕疵警報（紅色急速閃爍）。
- **Web Audio API 音效合成**：使用 OscillatorNode 生成 2200Hz 鋸齒波急促警報音，無需預載音檔。
- **可安裝為獨立 App**：支援 iOS Safari「加入主畫面」與 Android Chrome「安裝應用程式」，以全螢幕無框模式運行。

### 5.3 開發過程中的關鍵問題與解決方案

在系統整合過程中，共記錄了 14 項主要的技術問題與解決方案，以下摘錄最具代表性的幾項：

#### 5.3.1 ESP32 Core API 版本不相容

ESP32 Board Manager 從 Core 2.x 升級至 Core 3.x 後，LEDC PWM API 全面重構（`ledcSetup()` + `ledcAttachPin()` 改為 `ledcAttach()`），導致原始韌體無法編譯。解決方案為將所有 PWM 初始化程式碼更新為 Core 3.x 相容寫法。

#### 5.3.2 ESP32-CAM 拍攝瞬間電壓驟降 (Brownout)

ESP32-CAM 在高解析度模式下拍攝瞬間會抽取極大電流，導致 3.3V 供電不穩而崩潰。解決方案為在連線後立即發送 `SET_RES:1` 指令，強制降至 QVGA (320×240) 模式，在保證影像品質的同時避免供電問題。

#### 5.3.3 Domain Gap（領域落差）

最初以合成資料集訓練的 YOLO 模型在真實環境中出現大量漏判與誤判。解決方案為建立「收集 → 標註 → 微調」的閉環流程：系統內建一鍵拍照功能（S 鍵），搭配 labelImg 標註工具，再透過 `train_yolo.py` 進行 Fine-tuning，成功彌補了模擬與現實之間的差距。

#### 5.3.4 Serial DTR/RTS 控制導致 ESP32 重啟

Python 的 `pyserial` 開啟 Serial Port 時預設會拉低 DTR 與 RTS 腳位，在 ESP32 的電路設計中這等同於觸發硬體 Reset。解決方案為在開啟連線後立即執行 `setDTR(False)` 與 `setRTS(False)`，並等待 2.5 秒讓相機模組完成初始化。

### 5.4 系統特色與創新點

1. **多光譜融合降低誤判率**：不同於傳統單光源 AOI，本系統透過三色光譜依序掃描與通道融合技術，有效抑制因角度、色彩造成的假陽性。

2. **AI + OpenCV 雙模式自動切換**：系統啟動時自動偵測 AI 模型是否存在，若模型可用則使用 YOLOv8 進行精確辨識，否則無縫切換至 OpenCV 傳統演算法，確保系統在任何環境下都能運作。

3. **完全離線的行動監控系統**：ESP32S 以 AP 模式自建 Wi-Fi 熱點，手機連上後即可透過 WebSocket 接收即時數據。整套系統完全獨立於外部網路，適用於工廠無網路環境。

4. **內建 Fine-tuning 閉環流程**：從資料收集、標註到訓練，系統內建完整工具鏈，使用者無需機器學習專業知識即可針對特定場景進行模型優化。

5. **自動化 Excel 報表產出**：每次檢測自動生成含時間戳記的 `.xlsx` 報告，支援信心度高亮標記與九宮格面板編號換算，滿足品管追蹤需求。

## 6. 結論

本專題成功設計並實作了一套基於多光譜影像與 YOLOv8 深度學習的表面瑕疵智慧檢測系統。系統以低成本的 ESP32 系列微控制器為硬體核心，結合 Python 端 AI 運算能力，實現了從光源控制、影像擷取、瑕疵辨識、結果回饋到行動監控的完整檢測流程。

主要成果總結如下：

1. **硬體整合方面**：成功建構了 ESP32S（控制）、ESP32-CAM（影像）、PC（AI 運算）的三節點協作架構，透過自定義 Serial 協定實現穩定可靠的跨裝置通訊，解決了 DTR/RTS 控制、Brownout 供電等多項嵌入式系統整合難題。

2. **AI 檢測方面**：以 YOLOv8-nano 實現了 5 種瑕疵類型的即時辨識，推論時間低於 50 毫秒。透過遷移學習與 Fine-tuning 機制，成功解決了合成資料與真實場景之間的 Domain Gap 問題。

3. **系統完整度方面**：整合了 Wi-Fi AP 熱點 + WebSocket + PWA 手機 Dashboard，實現了低延遲（< 50ms）的無線行動監控；Excel 自動報表功能則滿足了品質管理的數據追蹤需求。

4. **實用性方面**：系統從操作手冊、標註工具整合、模型訓練腳本到除錯紀錄，均有完善的文件與自動化流程，具備良好的可維護性與可重製性。

本專題驗證了在有限預算下，透過嵌入式系統與開源 AI 工具的結合，可以建構出功能完善的自動化品質檢測系統，為中小型製造業的智慧化升級提供了可行的技術路徑參考。
