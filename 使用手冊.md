本手冊將指引你完成從硬體配線、單獨測試，到最終啟動 Python 整合系統的所有步驟。

## 1. 硬體配線指南

### ESP32S 控制器 (Controller)
*   **RGB LED (共陰極)**：
    *   R ➔ `GPIO 25`
    *   G ➔ `GPIO 26`
    *   B ➔ `GPIO 27`
    *   GND ➔ ESP32 `GND`
*   **I2C OLED 螢幕 (可選)**：
    *   SDA ➔ `GPIO 21`
    *   SCL ➔ `GPIO 22`
    *   VCC ➔ `3.3V` 或 `5V`
    *   GND ➔ `GND`
*   **無源蜂鳴器 (Passive Buzzer)**：
    *   正極 (長腳/貼紙側) ➔ `GPIO 4`
    *   負極 (短腳) ➔ `GND`
*   **實體觸發按鈕**：
    *   一端接 ➔ `GPIO 15` (程式內有啟動內建上拉電阻)
    *   一端接 ➔ `GND`

### ESP32-CAM 相機模組
*   **供電**：強烈建議接上 **`5V`** 腳位，若接 `3.3V` 極易發生拍照瞬間斷電當機的情況。
*   **燒錄模式接線 (使用 FTDI 時)**：`TX->U0R`, `RX->U0T`, **`GPIO 0 必須短路至 GND`**。
*   **執行模式接線**：**必須將 `GPIO 0` 與 `GND` 斷開**，並按下 RST 按鈕重新開機。

## 2. 單獨測試 (使用 Arduino Serial Monitor)

在進行綜合測試前，強烈建議先在 Arduino IDE 中分別驗證兩塊板子的功能。
⚠️ **測試條件**：Baud Rate 均為 `115200`，換行設定為 `New Line` 或 `Both NL & CR`。

**ESP32S 控制器指令：**
*   輸入 `LED:R`, `LED:G`, `LED:B`, `LED:ALL` 測試燈光。
*   輸入 `BUZZ:ON` / `BUZZ:OFF` 測試蜂鳴器。
*   按下實體按鈕，確認是否回傳 `BTN_PRESSED`。

**ESP32-CAM 相機指令：**
*   確認開機時是否有印出 `CAM_READY`。
*   輸入 `SET_RES:1` (設定為 QVGA，避免高畫質當機)。
*   輸入 `CAPTURE`，確認是否會噴出大量亂碼 (JPEG 檔案二進位資料) 並以 `IMG_OK` 結尾。

### 步驟 2：建立全新的虛擬環境


### 步驟 3：啟動虛擬環境

> ```

### 步驟 4：使用 requirements.txt 一鍵安裝所有套件

```powershell
pip install -r requirements.txt
```

| 套件 | 用途 |
|------|------|
| `pyserial` | 與 ESP32-CAM / ESP32S 的 Serial 通訊 |
| `opencv-python` | 影像處理與瑕疵檢測 |
| `numpy` | 數值與矩陣運算 |
| `openpyxl` | 匯出 Excel 瑕疵報告 |
| `ultralytics` | YOLOv8 AI 模型（訓練與推論），會自動安裝 PyTorch 等相依套件 |

> ⏱️ 安裝時間約 5~15 分鐘（視網路速度而定），`ultralytics` 會連帶安裝 PyTorch，檔案較大請耐心等候。

```powershell
python -c "import cv2; import serial; import ultralytics; print('所有套件安裝成功！')"
```
如果終端機印出 `所有套件安裝成功！`，代表環境已就緒。

### 步驟 6：啟動檢測系統

```powershell

```

## 4. 綜合測試與執行

### 啟動系統注意事項
1.  **重要前提**：執行 Python 程式前，**必須確保 Arduino IDE 的 Serial Monitor 已經完全關閉**，否則 Python 會噴出 `PermissionError(13, '存取被拒')` 錯誤。
2.  **確認 COM Ports**：記下 ESP32-CAM 與 ESP32S 的 COM 埠號碼（例如 CAM 是 COM3，Controller 是 COM4）。

### 啟動指令
在終端機中，導航至專案的 `pc_python` 目錄下，執行以下指令：
```bash
python defect_detector.py --cam COM3 --ctrl COM4
```

### 操作說明
程式啟動後，會彈出一個 OpenCV 的視窗介面。你可以在**該視窗**中按下以下快捷鍵（請確保鍵盤焦點在視窗上，而非終端機）：

*   **`空白鍵 (Space)` 或 點擊實體按鈕**：啟動全自動瑕疵檢測流程（紅光➔綠光➔藍光 依序拍攝並分析）。
*   **`R` / `G` / `B`**：切換為單一顏色的燈光並即時預覽（適合用來喬角度與確認打光）。
*   **`1`**：切換為單一面板模式 (預設)。
*   **`9`**：切換為 3×3 拼接面板模式 (針對大型物件的九宮格分割檢測)。
*   **自動產出報表**：每次完成全流程檢測 (`SPACE`) 或單色檢測 (`R/G/B`)，系統都會自動在 `inspection_reports/` 資料夾下生成一份帶有時間戳記的 Excel 瑕疵報告，標記所有的瑕疵位置、面積/編號與 AI 信心度。
*   **`S`**：儲存當前的拍攝畫面，影像會存放於專案底下的 `captured_images` 資料夾中。
*   **`Q` 或 `ESC`**：安全關閉系統，關閉燈光與連線。

## 5. AI 模型微調與資料收集 (Fine-tuning)

為解決真實環境的光影與角度落差（Domain Gap），系統內建了快速收集真實資料集的功能：

1. **收集照片**：在系統 GUI 畫面中按下 **`S`** 鍵，乾淨的原始影像會自動存入 `captured_images/` 資料夾。
2. **標註資料**：
   * 安裝：`pip install labelImg`
   * 開啟：在終端機輸入以下指令，以快速載入正確的 YOLO 類別與設定：
     ```bash
     labelImg captured_images captured_images\classes.txt captured_images
     ```
   * 將左方格式切換為 **YOLO**，使用 `W` 畫框，`Ctrl+S` 存檔，`D` 換下一張。
3. **重新訓練與模型運作方式**：
   * **自動搬移標註資料**：在 PowerShell 終端機（確保在 `pc_python` 目錄下）輸入以下指令，將圖片與標註檔分類移至訓練資料集中：
     ```powershell
     $src = "captured_images"
     $dest_img = "..\..\mini_led_defect_dataset_rgb_label_v2\train\images"
     $dest_lbl = "..\..\mini_led_defect_dataset_rgb_label_v2\train\labels"
     Move-Item -Path "$src\*.jpg" -Destination $dest_img -Force
     Get-ChildItem -Path $src -Filter "*.txt" | Where-Object { $_.Name -ne "classes.txt" } | Move-Item -Destination $dest_lbl -Force
     ```
   * **啟動訓練**：在終端機（確保已進入虛擬環境，且位於 `pc_python` 目錄下）輸入以下指令，程式會自動開始訓練：
     ```bash
     python train_yolo.py
     ```
   * **訓練方式 (Fine-tuning)**：程式會載入 YOLOv8 架構，並讀取 `dataset.yaml` 中定義的資料集。訓練過程中，AI 會結合先前的預訓練權重基礎，並學習您剛剛提供的新照片（包含真實環境的紙張紋理、反光、透視變形等特徵），進行 50 輪 (Epochs) 的微調學習。這能以最少的新資料量，極高效率地彌補模擬與現實的落差。
   * **運作方式 (Deployment)**：訓練結束後，系統會將表現最好的模型權重保存在 YOLO 預設的 `runs/` 資料夾下。而我們的 `train_yolo.py` 腳本會在最後一步，自動將最新生成的 `best.pt` 複製到 `pc_python/` 根目錄。
   * **執行最新模型**：您只需如同平常一樣，在終端機輸入啟動指令（請記得替換為您實際的 COM 埠號），檢測系統就會自動讀取剛訓練好的 `best.pt`，即刻生效：
     ```bash
     python defect_detector.py --cam COM3 --ctrl COM4
     ```

## 6. 無線行動監控 Dashboard 與手機警報系統使用說明

本系統內建 **PWA 無線行動監控 Dashboard**。無需下載安裝手機軟體，即可獲得與機台同步的閃光、聲音警報與詳盡的數據統計。

### 步驟 A：Arduino IDE 依賴庫安裝
在燒錄 ESP32S 韌體之前，必須先安裝對應的 WebSocket 通訊庫：
1. 開啟 Arduino IDE，點選 **「Tools (工具)」** ➔ **「Manage Libraries (管理庫)」**。
2. 搜尋 **`WebSockets`**。
3. 找到作者為 **`Markus Sattler`** 的開源庫（名稱就叫做 `WebSockets`，請注意**不要**安裝 Generic 或其他含有 generic 字眼的版本）。
4. 點選 **「Install (安裝)」**。
5. 安裝完後即可成功編譯並燒錄 `esp32s_controller.ino` 韌體。

### 步驟 B：手機連接 Wi-Fi 熱點
為了擺脫對任何外部路由器的依賴，ESP32S 開機後會主動發射自己的專用 Wi-Fi 訊號：
1. 打開手機的 Wi-Fi 設定。
2. 尋找並連線至：👉 **`Defect_Detector_WiFi`**。
3. 連線密碼為：👉 **`password123`**。

### 步驟 C：開啟手機端獨立 App (PWA)
1. 開啟手機瀏覽器（iOS 請用 Safari，Android 請用 Chrome）。
2. 在網址列輸入以下預設網址並前往：
   👉 **`http://192.168.4.1`** (或 `http://defect-detector.local`)
3. 畫面加載完成後，點擊網頁最底部的 **「立刻啟用」** 按鈕（這會向手機瀏覽器取得 Web Audio 播放權限，警報音方能正常工作）。
4. **💡 秘訣 (安裝成獨立 App)：**
   * **iOS (Safari)**：點擊分享按鈕 ➔ 選擇 **「新增至主畫面」**。
   * **Android (Chrome)**：點擊選單右上角三個點 ➔ 選擇 **「新增至主畫面」** 或 **「安裝應用程式」**。
   這會在手機桌面建立一個精美的 App 圖示。點擊它將會以**全螢幕、無瀏覽器外框**的原生 App 模式完美運行！

### 步驟 D：單色檢測與全功能連動
當手機與機台連線後，您可以隨時測試即時同步效果：
1. **單一顏色測試 (R/G/B)**：
   * 在 PC 端的 OpenCV 視窗按下鍵盤的 `R`、`G` 或 `B` 鍵。
   * PC 將指揮機台切換光源、拍攝照片並自動進行瑕疵檢測。
   * **檢測完成後，瑕疵數據會同步回傳給 ESP32S，機台的 OLED 螢幕會立刻印出總瑕疵數 (`Defects Count: X`) 以及主要瑕疵資訊，且手機端 App 也會在一瞬間同步響起急促的警報音並閃爍紅光顯示數據！**
2. **全流程檢測 (SPACE)**：
   * 按下機台實體按鈕或 PC 空白鍵啟動全功能多光譜掃描。
   * 掃描期間手機會顯示藍色呼吸燈跑馬燈同步。
   * 最終分析完畢，手機與機台將同時以極速回報最終的瑕疵分析與歷史紀錄。
