# 📌 感興趣方向 — 詳細展開

> ⚠️ **硬體平台說明：Arduino ESP32S**
>
> ESP32S 規格：雙核 240MHz、520KB SRAM、4MB Flash、支援 Wi-Fi / Bluetooth
> - ✅ **適合**：感測器資料擷取、光源/馬達控制、Wi-Fi 資料傳輸、簡易邊緣推論 (TinyML)
> - ❌ **不適合**：大型深度學習模型推論（YOLO、GAN、Transformer 等需要 GPU）
> - 💡 **建議架構**：**ESP32S（邊緣感測 + 控制）+ PC/伺服器（AI 模型訓練與推論）**，透過 Wi-Fi / Serial 通訊

---

## 一、瑕疵檢測方向

### 題目 A：基於機器視覺與光電感測之表面瑕疵智慧檢測系統

> **一句話說明**：用特定光源照射產品表面，AI 自動找出瑕疵位置與類型

**
產品可以是micro led螢幕 我自己畫一個RGB的紙板當作LCD螢幕 然後拿紅光、藍光、綠光LED照射，並在上面用黑筆畫一條線當作瑕疵，然後用ESP32-CAM拍攝影像，然後用AI辨識瑕疵位置座標與類型
1個的成功後換成數9個的拼接在一起(大量轉移)螢幕廠商最近在做的，然後用AI辨識瑕疵位置座標與類型

用LCD螢幕顯示瑕疵位置座標與類型，用蜂鳴器警示，同時用esp32的網路或藍芽功能，將瑕疵位置座標與類型傳送到PC端或手機(手機的話要app)
**

#### 🔧 系統架構（ESP32S 版本）
```
ESP32S 控制光源/傳送帶 → ESP32-CAM 拍攝影像 → Wi-Fi 傳送至 PC
→ PC 端 AI 瑕疵辨識 (YOLOv8/U-Net) → 結果回傳 ESP32S → OLED/蜂鳴器警示
```

#### 📋 Arduino ESP32S 硬體清單
| 元件 | 說明 | Arduino 相容型號 | 預估價格 |
|------|------|-----------------|---------|
| **ESP32S 主控板** | 主控制器，負責感測與通訊 | ESP32 DevKit V1 | ~150 元 |
| **ESP32-CAM** | 內建 OV2640 相機的 ESP32 模組 | AI-Thinker ESP32-CAM | ~200 元 |
| LED 環形光源 | 突顯表面瑕疵紋理 | WS2812B LED 環 / 白光 LED 環 | ~100 元 |
| OLED 顯示器 | 顯示檢測結果 | SSD1306 0.96" I2C OLED | ~60 元 |
| 蜂鳴器 | 瑕疵警報 | 有源蜂鳴器模組 | ~15 元 |
| 步進馬達 (選用) | 傳送帶模擬產線 | 28BYJ-48 + ULN2003 驅動板 | ~50 元 |
| 光敏電阻 (選用) | 環境光偵測，自動調整光源亮度 | GL5528 光敏電阻 | ~10 元 |

#### 🤖 AI 模型選擇
| 方法 | 模型 | 運行位置 |
|------|------|---------|
| 物件偵測 | YOLOv8-nano | 🖥️ PC 端 (Python) |
| 語意分割 | U-Net | 🖥️ PC 端 (Python) |
| 異常偵測 | AutoEncoder | 🖥️ PC 端 / ⚡ 簡化版可跑 ESP32 (TFLite Micro) |

#### 💡 ESP32S 角色
- 控制 LED 光源亮度與模式（PWM）
- ESP32-CAM 拍攝影像並透過 Wi-Fi 傳送至 PC
- 接收 PC 端 AI 辨識結果，驅動 OLED 顯示 + 蜂鳴器警示
- （選用）控制步進馬達移動工件

#### ⭐ 難度與亮點
- **難度**：⭐⭐⭐ (中等) — ESP32S 硬體整合難度適中
- **亮點**：結合光學設計與 AI，業界需求大（半導體、PCB、金屬加工）
- **ESP32S 可行性**：✅ **高** — ESP32-CAM 拍照 + Wi-Fi 傳輸方案成熟

### 題目 B：基於雷射光電感測與生成對抗網路 (GAN) 之工業零件 3D 瑕疵重建系統

> **一句話說明**：用雷射掃描零件取得 3D 資料，GAN 重建瑕疵並提出修復建議

#### 🔧 系統架構（ESP32S 版本）
```
ESP32S 控制雷射模組 + 步進馬達旋轉工件 → VL53L0X 測距取得輪廓資料
→ Wi-Fi 傳送至 PC → PC 端 GAN 3D 重建 → 結果回傳 ESP32S 顯示
```

#### 📋 Arduino ESP32S 硬體清單
| 元件 | 說明 | Arduino 相容型號 | 預估價格 |
|------|------|-----------------|---------|
| **ESP32S 主控板** | 主控制器 | ESP32 DevKit V1 | ~150 元 |
| 雷射模組 | 線雷射投影 | 650nm 紅光雷射模組 (KY-008) | ~30 元 |
| ToF 測距感測器 | 精確距離量測 | VL53L0X / VL53L1X (I2C) | ~120 元 |
| 步進馬達 | 旋轉工件取得各角度資料 | NEMA17 + A4988 驅動板 | ~150 元 |
| ESP32-CAM (選用) | 拍攝雷射線變形影像 | AI-Thinker ESP32-CAM | ~200 元 |
| OLED 顯示器 | 顯示掃描進度與結果 | SSD1306 0.96" I2C OLED | ~60 元 |

#### 🤖 AI 模型選擇
- **PointNet / PointNet++**：3D 點雲分類與分割 → 🖥️ PC 端
- **3D GAN**：生成修復後的 3D 模型 → 🖥️ PC 端 (需 GPU)
- **Pix2Pix**：2D 瑕疵影像修復 → 🖥️ PC 端

#### 💡 ESP32S 角色
- 控制雷射模組開關與亮度 (GPIO)
- 控制步進馬達精確旋轉工件 (A4988 驅動)
- 透過 VL53L0X 採集距離資料建構輪廓
- 資料透過 Wi-Fi 傳送至 PC 做 AI 推論

#### ⭐ 難度與亮點
- **難度**：⭐⭐⭐⭐ (較高) — 機電整合 + 3D 演算法
- **亮點**：3D + GAN 非常前沿，論文題目等級
- **ESP32S 可行性**：⚠️ **中等** — ESP32S 負責感測與控制OK，但 3D 重建必須在 PC 端

## 三、光電元件方向

### 題目 E：基於深度學習之有機發光二極體 (OLED) 材料性能預測與最佳化系統

> **一句話說明**：用 AI 預測 OLED 材料的發光效率，加速新材料篩選

#### 🔧 系統架構（ESP32S 版本）

```
ESP32S + 光譜/色度感測器量測 OLED 發光特性 → Wi-Fi 傳送至 PC
→ PC 端 AI 預測模型 (GNN/XGBoost) → 預測結果回傳 → ESP32S OLED 顯示
```

#### 📋 Arduino ESP32S 硬體清單
| 元件 | 說明 | Arduino 相容型號 | 預估價格 |
|------|------|-----------------|---------|
| **ESP32S 主控板** | 主控制器 | ESP32 DevKit V1 | ~150 元 |
| 光譜感測器 | 量測發光光譜 | AS7341 多通道光譜感測器 (I2C) | ~250 元 |
| 色度感測器 | 量測色溫/色度座標 | TCS34725 RGB 色度感測器 (I2C) | ~80 元 |
| 照度感測器 | 量測發光強度 | BH1750 照度感測器 (I2C) | ~30 元 |
| OLED 顯示器 | 顯示量測數據與預測結果 | SSD1306 0.96" I2C OLED | ~60 元 |

#### 🤖 AI 模型選擇
| 方法 | 模型 | 運行位置 |
|------|------|---------|
| 圖神經網路 | GNN | 🖥️ PC 端 |
| 傳統 ML | XGBoost / Random Forest | 🖥️ PC 端 / ⚡ 簡化版可跑 ESP32 |
| 序列模型 | Transformer (SMILES) | 🖥️ PC 端 (需 GPU) |

#### 💡 ESP32S 角色
- 透過 I2C 讀取光譜/色度/照度感測器數據
- 可控制 DAC 輸出不同驅動電流，量測 OLED 在不同條件下的表現
- 資料透過 Wi-Fi 上傳 PC 做 AI 分析
- 簡單的 XGBoost 模型可嘗試轉成 TFLite 在 ESP32S 上推論

### 題目 T：基於環境光感測與深度強化學習之建築節能智慧照明自動調控系統

> **一句話說明**：多個光感測器偵測室內各區域的自然光分佈，AI 用強化學習自動調控各區 LED 亮度，在維持舒適照度的同時最小化用電量

#### 🔧 系統架構（ESP32S 版本）
```
BH1750 照度感測器 x4 (各區域) → ESP32S 讀取各區照度
→ Q-Learning 最佳化各區 LED 亮度 → PWM 控制 LED 驅動
→ OLED 顯示節能數據 + Wi-Fi 上傳用電統計
```

#### 📋 Arduino ESP32S 硬體清單
| 元件 | 說明 | Arduino 相容型號 | 預估價格 |
|------|------|-----------------|---------|
| **ESP32S 主控板** | 主控制器 | ESP32 DevKit V1 | ~150 元 |
| 照度感測器 x4 | 各區域照度偵測 | BH1750 (I2C, 不同位址) | ~30 元 x4 |
| 高亮度 LED x4 | 各區域照明 | 大功率白光 LED + MOSFET 驅動 | ~40 元 x4 |
| 電流感測器 | 量測實際用電 | INA219 (I2C) | ~50 元 |
| PIR 人體感測器 | 偵測是否有人（無人關燈） | HC-SR501 x2 | ~50 元 |
| OLED 顯示器 | 顯示即時照度 + 節能百分比 | SSD1306 0.96" I2C OLED | ~60 元 |

#### 💡 ESP32S 角色
- I2C 讀取多個 BH1750 各區照度
- **Q-Learning 自動學習最佳亮度組合**（目標：照度 ≥ 500lux 且用電最低）
- PWM 控制各區 LED 亮度（0~100%）
- INA219 量測實際耗電, 計算節能百分比
- PIR 偵測無人時自動關燈
- OLED 顯示「已節省 XX% 電力」

- **類型**：♻️ **綠能節能 / 智慧建築**
- **展示方式**：做一個小型房間模型, LED 會自動隨窗戶光線調整, 螢幕顯示節省幾%電
